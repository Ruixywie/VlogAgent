# VlogAgent 实施计划

> 以论文发表为导向，从单视频美化（方向 A）做起，逐步扩展到多素材剪辑（方向 B）。

---

## 一、整体设计思路

### 1.1 核心范式

**完全对齐 PhotoAgent 的 Agent 化设计**——Perceiver → Planner → Executor → Evaluator 的闭环架构，迁移到视频领域。

**关键设计原则**：VlogAgent 不是一个"固定流水线 + 参数搜索器"，而是一个**真正的 Agent**——它自主感知视频问题、自主决定编辑策略、自主选择工具执行、自主评估结果。Agent 决定的不是"参数取多少"，而是"做什么 + 用什么工具 + 怎么做"。

同时，视频相比图片多出时间维度和音频维度，需要解决维度爆炸、动态美学量化、闭环成本三大问题。核心应对策略：**将视频问题转化为元数据/文本问题**，Agent 在符号空间完成规划与评估，只在执行阶段才接触实际视频流。

### 1.2 两阶段推进策略

```
阶段 A：单视频美化（基础模块）
  输入：一段原始视频（可能包含多镜头、多光照、多场景）
  输出：美化后的同一段视频
  价值：验证 MCTS + 闭环在视频上的可行性，积累视频美学评估经验
        ↓
        作为子模块嵌入
        ↓
阶段 B：多素材剪辑拼接（论文主体）
  输入：多段原始视频素材 + 简单主题描述
  输出：筛选、排序、拼接、美化后的完整 Vlog
  价值：核心创新（元数据空间 MCTS + 分解式奖励 + 分级代理评估）
```

### 1.3 论文定位

- **目标会议**：ACM Multimedia / AAAI / CVPR
- **暂定标题**：VlogAgent: Agentic Vlog Editing via Metadata-Space Planning and Tiered Proxy Evaluation
- **核心贡献**：
  1. 问题定义：首次定义"全自动多素材 Vlog 剪辑"任务
  2. 元数据空间 MCTS 规划
  3. 分解式奖励模型（5 维子指标）
  4. 分级代理评估策略（纯文本→Storyboard→Key Clips→完整渲染）

## 二、阶段 A：单视频美化模块（当前实现）

### 2.1 问题定义

用户提供**一段原始视频**（如手机拍摄的 1-5 分钟片段），这段视频可能包含：
- 多个镜头/场景（室内→室外，白天→傍晚）
- 不同区间的亮度/色温/曝光差异
- 局部抖动、失焦、噪点
- 无处理的原始音频

VlogAgent-A 需要自动对其进行美学增强，输出一段视觉统一、质量提升的视频。

### 2.2 系统架构：完整 Pipeline

### 2.3 核心设计决策

#### 2.3.1 搜索与最终执行分离（对齐 PhotoAgent）

PhotoAgent 在图片上直接操作内存像素（无损），VlogAgent 的视频编辑必须经过 H.264 编解码（有损）。解决方案：

- **搜索阶段**：直接在原视频上执行编辑方案（CRF=18），结果仅用于 Evaluator 评估，**最终全部丢弃**
- **最终执行**：确定完整 action_chain 后，在原始视频上一次性应用（CRF=10 视觉无损），**原始视频只被编码 1 次**
- **滤镜链合并**：多个可合并滤镜（color_adjust/denoise/sharpen/white_balance）合并为一条 FFmpeg `-vf` 链，避免多次编解码

#### 2.3.2 Perceiver 两阶段感知（参考 PhotoAgent）

原始设计将分析数据+关键帧一次性给 LLM 生成指令，LLM 容易做机械映射（"亮度低就提亮"），缺乏对画面内容的理解。

改进为两阶段：
- **阶段 1（观察）**：VLM 观看密集帧序列（每 2 秒 1 帧），以摄影师视角描述每段的画面内容、光线、色彩、构图、情绪、存在的问题
- **阶段 2（建议）**：将观察描述 + 技术指标一起给 VLM，基于对画面的**语义理解**生成编辑方案

效果：VLM 能理解"日落场景的暖色调是风格需要，不应纠正"等语义层面的判断。

#### 2.3.3 MCTS 搜索优化

- **VLM 看关键帧评估**：每段取首尾 2 帧（共约 8 帧），让 VLM 看着画面判断方案是否合理，而非纯文字预估
- **评估缓存**：相同动作组合的 VLM 评分缓存复用，15 次模拟实际 VLM 调用约 8-12 次
- **关键帧内容预构建**：搜索开始时构建一次关键帧内容，所有模拟共享

#### 2.3.4 Evaluator 权重设计（参考 PhotoAgent）

PhotoAgent 的评估以美学/感知质量为主导（ImageReward、LAION Aesthetic、UGC Reward 占主要权重），技术指标为辅。VlogAgent 对齐此思路：

| 维度 | 方法 | 权重 | 角色 |
|------|------|------|------|
| 整体美学 | VLM 逐段对比打分 | **60%** | 核心指标 |
| 内容保真 | CLIP 余弦相似度 | 15% | 防止过度修改 |
| 视觉质量 | Laplacian 方差 + 曝光检查 | 10% | 技术兜底 |
| 段间一致 | Lab 色彩距离 | 10% | 风格统一 |
| 音频完整 | 时长对比 | 5% | 音频保护 |

美学评分采用**逐段对比**：每个场景段取中间帧，原始 vs 编辑后并排展示给 VLM，确保每个场景都被覆盖。

#### 2.3.5 闭环累积机制

```
迭代1: Perceiver 观察原视频 → MCTS → 方案A → 评估超过基线 → 采纳
  action_chain = [A的动作]
  current_video = 原视频+A的结果

迭代2: Perceiver 观察"原视频+A" → MCTS → 方案B → 评估超过迭代1 → 采纳
  action_chain = [A的动作, B的动作]
  current_video = 原视频+A+B的结果

迭代3: 无提升 → 终止

最终执行: 原始视频 → 一次性应用 A+B 的完整滤镜链 → 输出
```

每轮 Perceiver 观察的是**累积编辑后的真实结果**，下一轮的建议是在此基础上的增量改进，方向不会偏离。

### 2.4 各模块详细设计

#### 2.4.1 视频分析（VideoAnalyzer）

**场景分割**：PySceneDetect ContentDetector（基于帧间差异的自适应阈值）

**逐段质量检测**（OpenCV，采样最多 30 帧）：
- 亮度：Y 通道均值 [0-255]
- 清晰度：Laplacian 方差
- 色温：R/B 通道比值估算
- 稳定性：光流场方差（calcOpticalFlowFarneback）
- 噪声：Laplacian 高频能量

**关键帧提取**：按时间间隔自适应（默认每 2 秒 1 帧，每段上限 10 帧），FFmpeg 精确跳转提取

**音频分析**：Whisper ASR 转录 + 语音区间检测

**输出数据模型**：
```python
@dataclass
class SegmentMetadata:
    seg_id: int
    time_range: tuple[float, float]
    mean_brightness: float       # Y通道均值 [0, 255]
    color_temp_est: float        # R/B 通道比值
    sharpness_score: float       # Laplacian 方差
    stability_score: float       # 光流方差（越小越稳）
    noise_level: float
    has_speech: bool
    speech_text: str
    keyframe_paths: list[str]    # 关键帧图片路径
```

#### 2.4.2 Perceiver（两阶段 VLM 感知）

**阶段 1 — 观察**：
- 输入：密集关键帧序列（每段标注时间范围）
- VLM 任务：以摄影师视角描述画面内容、光线、色彩、构图、情绪、问题
- 输出：自然语言观察描述

**阶段 2 — 建议**：
- 输入：阶段 1 的观察描述 + 技术指标
- VLM 任务：基于画面理解生成 8-15 条候选编辑指令
- 输出：`list[EditAction]`

**EditAction 数据模型**：
```python
@dataclass
class EditAction:
    action_description: str    # 语义描述（解释为什么要这样做）
    target_segment: str        # "seg-0" / "global"
    tool_type: str             # "basic" / "compound"
    tool_name: str             # "color_adjust" / "stabilize" / ...
    parameters: dict           # 工具参数
```

#### 2.4.3 Planner（MCTS 搜索）

**搜索参数**：depth=3, simulations=15, UCB_C=1.4, top_K=3

**MCTS 四阶段**：
1. **Selection**：UCB 选择最优路径
2. **Expansion**：展开一个未尝试的动作
3. **Simulation**：VLM 看每段首尾 2 帧 + 动作方案 → 预估效果分数
4. **Backpropagation**：回传奖励

**缓存机制**：动作组合签名（工具名+目标段+参数排序）作为 key，VLM 评分作为 value。15 次模拟中相同组合复用缓存，实际 VLM 调用 8-12 次。

**输出**：Top-3 EditPlan（有序动作序列 + 预估分）

#### 2.4.4 Executor（异构工具池）

**Basic Tools（FFmpeg 封装）**：

| 工具 | FFmpeg 实现 | 参数范围 | 可合并 |
|------|------------|---------|:------:|
| color_adjust | `eq=brightness:contrast:saturation:gamma` | brightness[-0.3,0.3] 等 | ✓ |
| white_balance | `colortemperature` | temperature[2000,10000] | ✓ |
| denoise | `hqdn3d` | strength[1,10] | ✓ |
| sharpen | `unsharp` | amount[0.5,3.0] | ✓ |
| stabilize | `vidstabdetect + vidstabtransform` | smoothing[5,30] | ✗ |
| speed_adjust | `setpts + atempo` | factor[0.25,4.0] | ✗ |
| apply_lut | `lut3d` | lut_file | ✗ |

**Compound Tools**：
- `auto_color_harmonize`：选锚点段（亮度中位数）→ 计算全局加权调整 → 单次 FFmpeg 编码（不拆分拼接）

**Tool Selector**：LLM 将语义指令翻译为具体工具名 + 参数

**编码策略**：
- 搜索阶段：`CRF=18, preset=fast`（速度优先，结果丢弃）
- 最终输出：`CRF=10, preset=slow`（视觉无损，只编码 1 次）

#### 2.4.5 Evaluator（5 维评估）

| 维度 | 方法 | 权重 | 详细 |
|------|------|------|------|
| **整体美学** | VLM 逐段对比 | 60% | 每段取中间帧，原始 vs 编辑后并排，VLM 从色彩/曝光/质量/一致性/美感 5 方面评分 |
| 内容保真 | CLIP 余弦相似度 | 15% | 原始/编辑各取 5 帧，OpenCLIP ViT-B-32 提取特征，计算平均余弦相似度 |
| 视觉质量 | Laplacian + 曝光 | 10% | 采样 20 帧，清晰度归一化 + 曝光范围 [80,180] 检查 |
| 段间一致 | Lab 距离 | 10% | 相邻段中间帧 Lab 欧氏距离，diff<20→满分，diff>60→零分 |
| 音频完整 | 时长对比 | 5% | 编辑前后时长比，变化<5%→满分，>30%→严重损坏 |

**基线评估**：闭环开始前先对原始视频打分，后续方案必须超过基线才被采纳。

### 2.5 各阶段 VLM 调用统计（3 轮迭代）

| 阶段 | 每轮调用 | 3轮合计 | 输入内容 |
|------|---------|---------|---------|
| Perceiver 观察 | 1 次 | 3 次 | 密集帧序列（~15-30帧） |
| Perceiver 建议 | 1 次 | 3 次 | 纯文字（观察描述+指标） |
| MCTS 搜索 | 8-12 次 | 24-36 次 | 每段首尾2帧（~8帧）+ 文字 |
| Evaluator 美学 | 3 次(Top-3) | 9 次 | 逐段对比帧（~8帧） |
| ToolSelector | ~6 次 | ~18 次 | 纯文字 |
| 基线评估 | — | 1 次 | 逐段对比帧 |
| **合计** | | **~60-70 次** | |

### 2.7 已解决的问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| VLM API 成本 | GPT-4o 昂贵 | 改用阿里云百炼 Qwen2.5-VL-72B（免费额度） |
| Perceiver 建议机械 | 只看数值，缺乏画面理解 | 两阶段感知（先观察再建议）+ 密集抽帧 |
| MCTS 评分无区分度 | 纯文字预估，所有方案得分相同 | VLM 看关键帧评估 + 缓存 |
| 多次编码画质损失 | 每步工具独立编码 | 搜索与最终执行分离 + 滤镜链合并 |
| 拼接点亮度跳变 | auto_color_harmonize 拆分拼接 | 改为全局滤镜单次编码，不拆分 |
| 锐化产生锯齿 | 在多次编码后的视频上锐化 | 最终执行只编码 1 次（CRF=10） |
| 初始分数为 0 | prev_score 未初始化 | 闭环前先评估原视频基线分数 |
| 一致性评分过严 | Lab 距离阈值太低 | 不同场景本身有色彩差异，放宽阈值 |
| Evaluator 过于死板 | 机械指标占 80% | 参考 PhotoAgent，美学占 60% |

### 2.8 下一步优化方向

1. **集成 DOVER**：替代 Laplacian 方差作为视觉质量评估
2. **段级编辑**：目前对非 global 指令也是全视频应用，需实现分段处理
3. **AI Tools 扩展**：物体移除（ProPainter）、天空替换、超分辨率等
4. **奖励模型训练**：MCTS 过程自动积累偏好对 → 训练 CLIP+MLP 替代高频 VLM 调用

## 三、阶段 B：多素材剪辑拼接（扩展设计）

### 3.1 在阶段 A 基础上的扩展

阶段 B 在阶段 A 的基础上增加三个新阶段，并将阶段 A 作为子模块嵌入：

```
多素材输入
   │
   ▼
┌──────────────────────┐
│  Stage 1: 素材理解     │  渐进式漏斗筛选（4 层 Layer）
│  (Material Analysis)   │
└──────────┬───────────┘
           │  素材卡片集合
           ▼
┌──────────────────────┐
```

### 3.2 阶段 B 新增的核心问题：多素材拼接后的视觉一致性

不同素材在不同时间、设备、环境下拍摄，即使各自美化后效果不错，拼在一起会出现亮度闪烁、色彩抖动、画质跳跃。

#### 问题分类

| 类型 | 表现 | 解决方向 |
|------|------|---------|
| 色彩空间差异 | 暖→冷的色彩突变 | 锚点 + Reinhard Color Transfer |
| 曝光/对比度差异 | 亮→暗的亮度跳变 | 全局亮度归一化 + 段间渐变 |
| 分辨率/画质差异 | 清晰→模糊的感知落差 | 统一分辨率或超分辨率 |
| 帧率差异 | 流畅度不一致 | FFmpeg fps 统一转换 |

#### 全局色彩统一策略

1. **锚点选择**：从所有素材中选 DOVER 评分最高段为参考
2. **Reinhard Color Transfer**：Lab 空间对齐（强度可调 α=0.6）
3. **段间过渡策略**：根据 CLIP 相似度自适应（硬切/短 crossfade/长 dissolve）

## 五、技术栈（当前实际选型）

### 视频分析 + 处理

| 组件 | 选型 | 用途 |
|------|------|------|
| 视频处理 | FFmpeg + OpenCV | 滤镜应用、裁切、色彩变换 |
| 场景分割 | PySceneDetect (ContentDetector) | 镜头边界检测 |
| 语音识别 | Whisper (base) | ASR 转录 + 语音区间检测 |
| 色彩统一 | FFmpeg eq + colortemperature | 全局滤镜（不拆分视频） |

### Agent 核心组件

| 组件 | 选型 | 用途 |
|------|------|------|
| Perceiver / Evaluator (VLM) | Qwen2.5-VL-72B (阿里云百炼) | 画面观察 + 编辑建议 + 美学评分 |
| Planner (MCTS) | 自研（参考 PhotoAgent） | 搜索最优动作序列 |
| Tool Selection | LLM 驱动 | 语义指令 → 工具参数 |
| 内容保真度 | OpenCLIP ViT-B-32 | CLIP 余弦相似度 |

### Evaluator（5 维分解评估）

| 评估维度 | 模型/方法 | 权重 | 是否需要训练 |
|---------|----------|------|:---:|
| **整体美学** | VLM 逐段对比 (Qwen-VL) | **60%** | 否 |
| 内容保真度 | CLIP 余弦相似度 | 15% | 否 |
| 视觉质量 | Laplacian 方差 + 曝光检查 | 10% | 否 |
| 段间一致性 | Lab 色彩距离 | 10% | 否 |
| 音频完整性 | 时长对比 | 5% | 否 |

后续阶段：
- 视觉质量升级为 DOVER
- 编辑美学训练 CLIP+MLP 奖励模型（MCTS 过程自动积累偏好对）

## 六、风险与缓解

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|---------|
| MCTS 在动作序列空间收敛慢 | 中 | 延期 | VLM 看关键帧提升评估质量 + 缓存减少调用；备选：贪心搜索 |
| 段间一致性处理效果不佳 | 中 | 质量 | auto_color_harmonize 单次编码避免拼接跳变；Perceiver 自主发现 |
| 美学评估主观性导致实验方差大 | 中 | 论文 | 增加评估者数量 (20+)；报告 Fleiss' Kappa；多组 case study |
| VLM API 成本过高 | 中 | 预算 | 阿里云百炼免费额度；MCTS 缓存减少调用；后续训练奖励模型 |
| 多次编码画质损失 | 低 | 质量 | 搜索执行分离 + 滤镜链合并 + CRF=10 视觉无损 |
| AI 工具生成质量不稳定 | 中 | 质量 | Evaluator 自动检测质量下降 → 回退；优先 Basic Tools |
