### 1. PhotoAgent: Agentic Photo Editing with Exploratory Visual Aesthetic Planning

| 属性 | 信息 |
|------|------|
| **作者** | Mingde Yao, Zhiyuan You, King-Man Tam, Menglu Wang, Tianfan Xue |
| **年份** | 2025 |
| **链接** | [arXiv:2602.22809](https://arxiv.org/abs/2602.22809) |
| **与 VlogAgent 的关联** | 核心范式来源：MCTS 美学规划 |

#### 主要内容

PhotoAgent 将自主图像编辑建模为**长时域决策问题（Long-Horizon Decision-Making）**，提出了一个三阶段的智能体系统：

1. **美学推理（Aesthetic Reasoning）**：MLLM 解读用户上传照片的视觉审美意图，无需用户提供逐步编辑指令
2. **MCTS 规划（Exploratory Planning）**：采用蒙特卡洛树搜索探索多步编辑操作的最优序列
   - 每个节点代表一个编辑状态（当前图片 + 已执行操作历史）
   - 每条边代表一个编辑动作（调整亮度、对比度、套用滤镜等）
   - 通过降分辨率 rollout 大幅降低计算开销：MCTS 搜索阶段使用低分辨率图片进行快速评估，确定最优路径后再以全分辨率执行
3. **闭环执行（Closed-Loop Execution）**：结合记忆机制和视觉反馈，Agent 执行编辑后由视觉模型检查效果，不满意则撤回重做

同时引入了 **UGC-Edit 基准**：7,000 张用户生成照片 + 学习式美学奖励模型，用于系统性评估。在 1,017 张照片的测试集上，指令遵循和视觉质量均优于基线方法。

#### 对 VlogAgent 的关键借鉴点

| # | 借鉴点 | 如何迁移到 VlogAgent |
|---|--------|---------------------|
| 1 | **MCTS 树搜索规划范式** | VlogAgent 的核心规划引擎。但需将搜索空间从"图片编辑操作序列"改为"片段选择与排序的元数据方案"，在符号空间而非视频流上搜索 |
| 2 | **降分辨率 rollout 策略** | 直接迁移为"关键帧拼图（Storyboard Proxy）"机制：用抽帧拼图替代视频渲染进行快速评估 |
| 3 | **学习式美学奖励模型** | 需要构建 Vlog 专用版本。PhotoAgent 的单图美学评分可作为 VlogAgent 分解式奖励中"单帧视觉质量"子指标的基础 |
| 4 | **闭环反馈 + 记忆机制** | 迁移为"三层闭环架构"（文本层→Storyboard 层→视频层），避免直接在视频流上做闭环的高昂代价 |
| 5 | **自主美学意图理解** | VlogAgent 同样需要从用户提供的简单主题描述中推理出编辑风格偏好 |

### 1.5 PhotoArtAgent: Intelligent Photo Retouching with Language Model-Based Artist Agents

| 属性 | 信息 |
|------|------|
| **年份** | 2025 |
| **链接** | [arXiv:2505.23130](https://arxiv.org/abs/2505.23130) |
| **OpenReview** | 被拒（评分 2/2/4/2） |
| **与 VlogAgent 的关联** | VLM + 专业工具 API 的修图范式 |

#### 主要内容

PhotoArtAgent 用 GPT-4o 驱动 Adobe Lightroom API 进行照片修图，无需训练。三阶段流程：图像分析与策略提案 → 参数生成（JSON 输出 Lightroom 参数）→ 反思循环（76.2% 的图像需多轮迭代）。用户研究评分 6.50/10，超过两位人类专家。

#### 被拒教训（对 VlogAgent 的重要警示）

| 审稿批评 | VlogAgent 如何避免 |
|---------|-------------------|
| "贡献就是 prompt engineering" | VlogAgent 有 CLIP+MLP 训练组件，计划 GRPO 训练 |
| "编辑任务太基础（只有全局调整）" | VlogAgent 有段级编辑 + Stage-Aware Planning |
| "Lightroom Auto 一秒搞定效果差不多" | 需要和 FFmpeg auto-levels 等简单基线正面对比 |
| "没有和 MonetGPT 等有训练的方法比" | 需要完整的基线对比 |

### 2. PersonaVlog: Personalized Multimodal Vlog Generation with Multi-Agent Collaboration and Iterative Self-Correction

| 属性 | 信息 |
|------|------|
| **作者** | Xiaolu Hou, Bing Ma, Jiaxiang Cheng, Xuhua Ren, Kai Yu, Wenyue Li, Tianxiang Zheng, Qinglin Lu |
| **年份** | 2025 |
| **链接** | [arXiv:2508.13602](https://arxiv.org/abs/2508.13602) |
| **与 VlogAgent 的关联** | 最直接相关：多智能体 Vlog 生成 |

#### 主要内容

PersonaVlog 是目前与 VlogAgent 设想**最接近的已有工作**，但定位有本质差异——它是**生成式**的（从主题生成全新视频内容），而非**编辑式**的（对已有素材进行剪辑）。

核心架构：
1. **多智能体协作框架（MACF）**：模拟人类制作团队，设置多个专业化 Agent 分工协作
   - **Story Agent**：根据主题和参考图像生成故事脚本
   - **Storyboard Agent**：将脚本分解为分镜头方案
   - **Video Description Agent**：为每个镜头生成详细的视频描述提示词
   - **Monologue Agent**：生成角色独白/旁白文本
   - **Music Description Agent**：生成背景音乐描述提示词
2. **反馈回滚机制（FRM）**：MLLM 作为自动审阅者，对生成的关键帧和视频进行评估
   - 如果生成结果不满足质量要求，自动回滚到上一步并重新生成
   - 实现了迭代式的自我修正（Iterative Self-Correction）
3. **ThemeVlogEval**：主题式自动化评测框架，提供标准化评估指标和数据集

#### 对 VlogAgent 的关键借鉴点

| # | 借鉴点 | 如何迁移到 VlogAgent |
|---|--------|---------------------|
| 1 | **多智能体分工模式** | 可参考其角色分工思路，但 VlogAgent 的 Agent 职责不同：素材分析 Agent、叙事规划 Agent、编辑执行 Agent、美学评审 Agent |
| 2 | **反馈回滚机制（FRM）** | 直接借鉴：VlogAgent 的 Critic Agent 发现问题后，可指定回滚到具体阶段重做，而非从头开始 |
| 3 | **ThemeVlogEval 评测范式** | 可参考其评测维度（主题相关性、视觉质量、音频匹配度），适配到 VlogAgent 的编辑质量评估场景 |
| 4 | **音乐描述生成** | VlogAgent 的 BGM 选配可借鉴其"先用文本描述所需音乐风格，再检索/生成匹配音乐"的流程 |

**关键差异警示**：PersonaVlog 依赖视频生成模型（如 Sora 类）创造新内容，VlogAgent 操作的是用户已有的真实素材。这意味着 VlogAgent 不需要处理"生成一致性"问题，但需要处理"素材筛选与理解"问题。

### 3. EditDuet: A Multi-Agent System for Video Non-Linear Editing

| 属性 | 信息 |
|------|------|
| **作者** | Marcelo Sandoval-Castaneda, Bryan Russell, Josef Sivic, Gregory Shakhnarovich, Fabian Caba Heilbron (Adobe Research) |
| **年份** | 2025 |
| **链接** | [arXiv:2509.10761](https://arxiv.org/abs/2509.10761) |
| **发表** | SIGGRAPH 2025 |
| **与 VlogAgent 的关联** | Editor-Critic 双智能体架构参考 |

#### 主要内容

EditDuet 专注于视频**非线性编辑（NLE）**任务，特别是 A-roll / B-roll 的编排。核心创新在于其双智能体交互设计：

1. **Editor Agent**：
   - 接收视频片段集合和自然语言编辑指令
   - 使用标准视频编辑软件工具（裁剪、拼接、调整时间线等）生成编辑序列
   - 将编辑过程建模为**序列决策问题**
2. **Critic Agent**：
   - 审阅 Editor 的编辑结果
   - 提供自然语言反馈（具体的改进建议）或批准最终渲染
   - 两个 Agent 之间通过**学习式通信协议**进行交互，而非简单的 prompt 传递
3. **LLM-as-Judge 评估指标**：提出基于 LLM 的编辑质量评估方法

实验结果：失败率仅 8.2%，时间覆盖率 89.8%，在用户研究中显著优于现有方法。

#### 对 VlogAgent 的关键借鉴点

| # | 借鉴点 | 如何迁移到 VlogAgent |
|---|--------|---------------------|
| 1 | **Editor-Critic 双 Agent 架构** | 直接采用：VlogAgent 的编辑执行模块（Editor）和美学评估模块（Critic）可沿用此模式 |
| 2 | **学习式智能体间通信** | 比简单的 prompt 传递更高效。VlogAgent 可训练 Agent 间的结构化通信协议，减少无效迭代 |
| 3 | **基于 NLE 时间线的操作抽象** | EditDuet 使用标准 NLE 工具集（裁剪、拼接等），VlogAgent 的 Editor Agent 工具集设计可直接参考 |
| 4 | **LLM-as-Judge 评估** | VlogAgent 的冷启动阶段可直接使用 LLM-as-Judge 作为美学评估的 baseline |
| 5 | **编辑过程的序列决策建模** | 与 PhotoAgent 的决策建模思路一致，验证了将视频编辑建模为序列决策问题的可行性 |

### 4. LAVE: LLM-Powered Agent Assistance and Language Augmentation for Video Editing

| 属性 | 信息 |
|------|------|
| **作者** | Bryan Wang, Yuliang Li, Zhaoyang Lv, Haijun Xia, Yan Xu, Raj Sodhi |
| **年份** | 2024 |
| **链接** | [arXiv:2402.10294](https://arxiv.org/abs/2402.10294) |
| **发表** | ACM IUI 2024 |
| **与 VlogAgent 的关联** | 素材语言标注与语义驱动编辑 |

#### 主要内容

LAVE 的核心思想是**用语言作为视频编辑的桥梁**：先自动为视频素材生成文本描述，然后用户通过自然语言指令驱动编辑操作。

系统功能：
1. **自动语言标注**：系统自动分析用户的视频素材，为每个片段生成语言描述（内容、场景、动作等），建立视频-文本的对应关系
2. **语义检索**：用户可通过自然语言搜索素材（如"找到有日落的片段"）
3. **故事板排序（Storyboarding）**：基于语义理解，自动或半自动地将片段排列成连贯的叙事序列
4. **智能裁剪**：根据语义上下文自动确定片段的最佳起止点
5. **混合交互模式**：支持 Agent 辅助自动编辑 + 用户手动 UI 操作的灵活切换

用户研究：8 名参与者（从新手到专业编辑），验证了 LAVE 在降低编辑门槛方面的有效性。

#### 对 VlogAgent 的关键借鉴点

| # | 借鉴点 | 如何迁移到 VlogAgent |
|---|--------|---------------------|
| 1 | **"语言作为视频的代理表示"核心思想** | 这与 VlogAgent 的统一核心洞察完全一致——将视频问题转化为文本/元数据问题。LAVE 验证了这条路径的可行性 |
| 2 | **素材自动语言标注流程** | VlogAgent Stage 1（素材理解）的直接参考：自动为每段素材生成结构化文本描述 |
| 3 | **语义检索能力** | VlogAgent 在叙事规划阶段需要按语义检索候选片段，LAVE 的检索机制可复用 |
| 4 | **Storyboarding 排序逻辑** | VlogAgent Stage 2（叙事规划）的重要参考 |
| 5 | **人机协作交互设计** | VlogAgent 的"人机协作模式"可参考 LAVE 的混合交互设计 |

**特别价值**：LAVE 是唯一通过用户研究验证了"语言驱动视频编辑"端到端可行性的工作，为 VlogAgent 的"元数据空间操作"策略提供了实证支撑。

### 5. VideoAgent: Long-form Video Understanding with Large Language Model as Agent

| 属性 | 信息 |
|------|------|
| **作者** | Xiaohan Wang, Yuhui Zhang, Orr Zohar, Serena Yeung-Levy |
| **年份** | 2024 |
| **链接** | [arXiv:2403.10517](https://arxiv.org/abs/2403.10517) |
| **与 VlogAgent 的关联** | 智能体式视频理解方法 |

#### 主要内容

VideoAgent 提出了一种**以 LLM 为中央决策智能体**的长视频理解框架，核心创新在于"选择性地看"而非"全部看完"：

1. **LLM 中央 Agent**：LLM 不直接处理视频像素，而是作为推理和规划的中枢
2. **工具调用架构**：VLM（视觉-语言模型）和 CLIP 作为 Agent 可调用的"视觉工具"
   - Agent 根据当前推理需要，决定"接下来需要看哪些帧"
   - 工具返回视觉信息（帧描述、物体识别等），Agent 整合后继续推理
3. **迭代式信息搜索**：Agent 不断提出子问题 → 定位相关帧 → 获取信息 → 判断是否足够回答原始问题
4. **极致效率**：平均仅需 **~8 帧** 即可理解长视频，在 EgoSchema（54.1%）和 NExT-QA（71.3%）上达到零样本 SOTA

#### 对 VlogAgent 的关键借鉴点

| # | 借鉴点 | 如何迁移到 VlogAgent |
|---|--------|---------------------|
| 1 | **选择性帧采样策略** | VlogAgent 素材理解阶段的核心策略：不需要处理素材的每一帧，而是由 Agent 动态决定需要"看"哪些关键帧 |
| 2 | **LLM-as-Agent + VLM-as-Tool 架构** | VlogAgent 可直接采用：LLM 负责规划和推理，VLM 按需提供视觉信息 |
| 3 | **迭代式信息聚合** | 素材理解阶段：Agent 先粗览，发现信息不足时再深入查看特定片段的更多帧 |
| 4 | **低帧数高效理解的实证** | 证明了"少量关键帧 + LLM 推理"足以理解视频内容，支撑了 VlogAgent "关键帧拼图"策略的理论基础 |

### 6. StoryAgent: Customized Storytelling Video Generation via Multi-Agent Collaboration

| 属性 | 信息 |
|------|------|
| **作者** | Panwen Hu, Jin Jiang, Jianqi Chen, Mingfei Han, Shengcai Liao, Xiaojun Chang, Xiaodan Liang |
| **年份** | 2024 |
| **链接** | [arXiv:2411.04925](https://arxiv.org/abs/2411.04925) |
| **与 VlogAgent 的关联** | 多智能体叙事规划 |

#### 主要内容

StoryAgent 解决"叙事视频生成"中的**跨镜头主体一致性**问题：

- **多 Agent 分工**：Story Design Agent（剧情设计）→ Storyboard Agent（分镜生成）→ Video Creation Agent（视频生成），每个阶段由专业化 Agent 负责
- **LoRA-BE 技术**：一种定制化的图像到视频方法，增强镜头内的时间一致性（intra-shot temporal consistency）
- **分镜生成管线**：维持镜头间的主体外观一致性

#### 对 VlogAgent 的关键借鉴点

- **叙事结构规划的多 Agent 分解方式**：StoryAgent 将"故事→分镜→视频"的创作流程拆分给不同 Agent，VlogAgent 的"素材理解→叙事规划→编辑执行→美学评估"流水线可参考这种分工粒度
- **分镜（Storyboard）作为中间表示**：StoryAgent 使用分镜作为从文本叙事到视频的中间桥梁，这与 VlogAgent 的"关键帧拼图"机制异曲同工

### 7. VideoDirectorGPT: Consistent Multi-Scene Video Generation via LLM-Guided Planning

| 属性 | 信息 |

| **作者** | Han Lin, Abhay Zala, Jaemin Cho, Mohit Bansal |
| **年份** | 2024 |
| **链接** | [arXiv:2309.15091](https://arxiv.org/abs/2309.15091) |
| **发表** | COLM 2024 |
| **与 VlogAgent 的关联** | LLM 引导的场景规划 |

#### 主要内容

VideoDirectorGPT 利用 LLM 的规划能力实现**多场景长视频的一致性生成**：

1. **Video Planner**：使用 GPT-4 将用户的单句文本提示展开为详细的视频规划方案

2. **Layout2Vid Generator**：基于规划方案进行布局控制的视频生成
   - 提供显式的空间布局控制

#### 对 VlogAgent 的关键借鉴点

- **LLM 作为"导演"进行场景级规划**：VideoDirectorGPT 证明了 LLM 有能力生成结构化的多场景规划方案，VlogAgent 的叙事规划模块可借鉴其"文本提示→结构化场景方案"的展开逻辑
- **规划方案的结构化输出格式**：场景描述 + 实体布局 + 一致性约束的 JSON 结构，可作为 VlogAgent 编辑方案 JSON 格式的参考

### 8. Video Quality Assessment: A Comprehensive Survey

| 属性 | 信息 |
|------|------|
| **作者** | Qi Zheng, Yibo Fan, Leilei Huang 等 |
| **年份** | 2024 |
| **链接** | [arXiv:2412.04508](https://arxiv.org/abs/2412.04508) |
| **与 VlogAgent 的关联** | 视频质量评估综述 |

#### 主要内容

全面综述视频质量评估（VQA）领域的最新进展：

- **传统方法的局限**：基于自然图像/视频统计的传统 VQA 模型在 UGC 内容上效果有限
- **深度学习方法**：CNN/Transformer 基础的端到端 VQA 模型（如 DOVER、FAST-VQA）
- **大型多模态模型方法**：利用 LMM 进行视频质量感知和评估
- **基准数据集**：DIVIDE-3k、KoNViD-1k、YouTube-UGC 等
- **开放问题**：UGC 视频的质量评估仍是挑战，特别是主观美学与技术质量的分离

#### 对 VlogAgent 的关键借鉴点

- **DOVER 等 VQA 模型**：可直接用于 VlogAgent 分解式奖励中的"单帧/片段视觉质量"子指标
- **美学 vs 技术质量的分离**：VlogAgent 的奖励模型需要区分"技术质量"（清晰度、稳定性）和"编辑美学"（叙事、节奏），该综述提供了理论框架
- **UGC 场景的特殊挑战**：VlogAgent 处理的用户素材本质上就是 UGC 内容，需要针对 UGC 特点设计质量评估策略

### 10. Generative AI for Film Creation: A Survey of Recent Advances

### 11. FunClip: Open-source Video Speech Recognition & Clipping Tool

| 属性 | 信息 |
|------|------|
| **来源** | ModelScope（阿里达摩院） |
| **链接** | [GitHub](https://github.com/modelscope/FunClip) |
| **与 VlogAgent 的关联** | 语音驱动的剪辑工具 |

#### 主要内容

FunClip 是一个开源的**基于语音识别的智能视频剪辑工具**：

- **核心功能**：使用 ASR（语音识别）自动转录视频中的语音内容，然后基于文本内容进行智能裁剪
- **工作流**：视频 → Whisper/FunASR 语音识别 → 文本时间戳对齐 → 基于关键词/语义的片段筛选 → 自动裁剪输出
- **特点**：支持中英文混合识别、说话人分离、热词定制

#### 对 VlogAgent 的关键借鉴点

- **语音驱动的剪辑点检测**：对于包含大量语音的 Vlog 素材（如旅行解说、美食评测），语音内容是确定剪辑点的重要信号。VlogAgent 的素材理解阶段可集成类似能力
- **ASR + 时间戳对齐**：精确的语音-时间码对应关系是 VlogAgent 生成素材卡片的关键数据，FunClip 的实现方案可直接复用
- **可直接集成的开源工具**：作为 VlogAgent Stage 1（素材理解）中语音处理模块的候选实现

### 12. VideoExplorer: Think With Videos For Agentic Long-Video Understanding

| 属性 | 信息 |
|------|------|
| **作者** | Huaying Yuan, Zheng Liu, Junjie Zhou, Hongjin Qian, Yan Shu, Nicu Sebe, Ji-Rong Wen, Zhicheng Dou |
| **年份** | 2025 |
| **链接** | [arXiv:2506.10821](https://arxiv.org/abs/2506.10821) |
| **代码仓库** | [GitHub: VideoDeepResearch](https://github.com/yhy-2000/VideoDeepResearch) |
| **与 VlogAgent 的关联** | 智能体式长视频分析 |

> **注意**：可行性报告中将此论文标注为"VideoDeepResearch"，实际论文标题为"VideoExplorer"，"VideoDeepResearch"是其 GitHub 代码仓库名称。

#### 主要内容

VideoExplorer 提出了一种"**用视频思考（Think With Videos）**"的智能体式长视频理解框架：

1. **迭代推理机制**：框架迭代地制定子问题 → 时间定位相关片段 → 执行面向任务的视频理解
2. **统一推理流程**：将规划（Planning）、时间定位（Temporal Grounding）和感知（Perception）融入统一的推理循环
3. **两阶段训练**：
   - 监督轨迹初始化（Supervised Trajectory Initialization）
   - 轨迹级偏好优化（Trajectory-level Preference Optimization）
4. **难度自适应采样**：构建长视频推理数据集时，采用难度自适应策略确保复杂任务的高质量训练轨迹

特点：实现了忠实（faithful）、高效（efficient）且可解释（interpretable）的推理能力。

#### 对 VlogAgent 的关键借鉴点

- **迭代式子问题分解**：VlogAgent 的素材理解阶段可采用类似策略——将"理解这段素材讲了什么"分解为多个子问题（场景是什么？谁在其中？发生了什么事件？情绪如何？），逐步聚合信息
- **时间定位能力**：VideoExplorer 的自适应时间定位机制可用于 VlogAgent 精确定位素材中的"高光时刻"或"关键情节点"
- **轨迹级偏好优化**：这种训练方法论可用于训练 VlogAgent 的叙事规划模块——将不同的编辑方案视为"轨迹"，通过偏好优化学习更好的编辑决策
